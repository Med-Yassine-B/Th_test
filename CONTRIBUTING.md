# Contributing to Th_test
First, Thansk for your interrest in contributing to **Th_test**

## How to contribute

### issues
- opening an issue should be clear and easy to understand
### Pull requests
- Pull Requests should be related to an issue (so simply to modefy something open an issue and ask a maintainer to assign it to u then u can safely start working on it!)
- Pull Requests should include a "Closes #<issue number>" to close the issue automatically
### commits
- commits should be simple and clear and follow this format fix(#<issues number>):<commit message>
- commits can be labeled with diffrent labels chore|fix|feature|refactor
- commits should change one thing at a time

## Code style
- keep things minimalist and evrything should fit in a single header file
- no dependencyes (only built in libraries <vector> <iostream> ...)
- make sure the code compiles before making a PR
